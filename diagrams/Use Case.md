# Глоссарий

| Термин | Определение |
|:--|:--|
| Пользователь | Человек, использующий приложение для заказа пиццы |
| Гость | Пользователь, который использует приложение без регистрации |
| Зарегистрированный пользователь | Пользователь с учётной записью в приложении |
| Администратор | Управляет меню и заказами |
| Заказ | Сформированная корзина с выбранными товарами и адресом доставки |
| Корзина | Содержимое выбранных товаров перед оформлением заказа |
| Промокод | Код, который можно применить для получения скидки |

## 1 Актёры

| Актёр | Описание |
|:--|:--|
| Guest | Посетитель приложения, который может просматривать меню и добавлять товары в корзину без регистрации |
| RegisteredUser | Зарегистрированный пользователь, который может просматривать меню, управлять корзиной, оформлять заказы и редактировать профиль |
| Admin | Администратор, который управляет меню и заказами |

## 2 Варианты использования

### 2.1 Просмотреть меню (Browse Menu)

**Актёры:** Guest, RegisteredUser  
**Описание:** Позволяет пользователю просмотреть список товаров/меню.  
**Предусловия:** Пользователь открыл приложение.  
**Основной поток:**
1. Пользователь выбирает функцию просмотра меню;
2. Приложение отображает все доступные товары;
3. Вариант использования завершается.

---

### 2.2 Добавить в корзину (Add to Cart)

**Актёры:** Guest, RegisteredUser  
**Описание:** Позволяет пользователю добавить выбранный товар в корзину.  
**Предусловия:** Пользователь просматривает меню.  
**Основной поток:**
1. Пользователь выбирает товар;
2. Пользователь добавляет товар в корзину;
3. Вариант использования завершается.

---

### 2.3 Зарегистрироваться (Register)

**Актёры:** Guest  
**Описание:** Позволяет гостю создать учётную запись.  
**Предусловия:** Пользователь выбрал функцию регистрации.  
**Основной поток:**
1. Приложение отображает окно регистрации;
2. Пользователь вводит имя и подтверждает;
3. Приложение создаёт новый профиль пользователя;
4. Вариант использования завершается.

---

### 2.4 Войти в систему (Login)

**Актёры:** Guest  
**Описание:** Позволяет гостю войти в учётную запись.  
**Предусловия:** Пользователь выбрал функцию входа.  
**Основной поток:**
1. Приложение отображает окно входа;
2. Пользователь вводит данные для входа;
3. Приложение проверяет данные и присваивает статус "RegisteredUser";
4. Вариант использования завершается.

---

### 2.5 Редактировать корзину (Edit Cart)

**Актёры:** RegisteredUser  
**Описание:** Позволяет пользователю изменять содержимое своей корзины.  
**Предусловия:** Пользователь добавил товары в корзину.  
**Основной поток:**
1. Пользователь открывает корзину;
2. Пользователь изменяет количество товаров или удаляет их;
3. Вариант использования завершается.

---

### 2.6 Оформить заказ (Place Order)

**Актёры:** RegisteredUser  
**Описание:** Позволяет зарегистрированному пользователю оформить заказ.  
**Предусловия:** Корзина пользователя содержит товары.  
**Основной поток:**
1. Пользователь подтверждает заказ;
2. Приложение рассчитывает итоговую сумму и создаёт заказ;
3. Вариант использования завершается.

---

### 2.7 Отслеживать заказ (Track Order)

**Актёры:** RegisteredUser  
**Описание:** Позволяет пользователю отслеживать статус своего заказа.  
**Предусловия:** Пользователь оформил заказ.  
**Основной поток:**
1. Пользователь выбирает заказ для отслеживания;
2. Приложение отображает текущий статус заказа;
3. Вариант использования завершается.

---

### 2.8 Редактировать профиль (Edit Profile)

**Актёры:** RegisteredUser  
**Описание:** Позволяет пользователю изменять данные своего профиля.  
**Предусловия:** Пользователь вошёл в систему.  
**Основной поток:**
1. Пользователь открывает окно редактирования профиля;
2. Пользователь изменяет данные и сохраняет;
3. Вариант использования завершается.

---

### 2.9 Выйти из системы (Logout)

**Актёры:** RegisteredUser  
**Описание:** Позволяет пользователю выйти из учётной записи.  
**Предусловия:** Пользователь вошёл в систему.  
**Основной поток:**
1. Пользователь выбирает функцию выхода;
2. Приложение завершает сеанс пользователя;
3. Вариант использования завершается.

---

### 2.10 Управлять меню (Manage Menu)

**Актёры:** Admin  
**Описание:** Позволяет администратору добавлять, удалять и изменять товары в меню.  
**Предусловия:** Администратор вошёл в систему.  
**Основной поток:**
1. Администратор открывает управление меню;
2. Администратор вносит изменения;
3. Вариант использования завершается.

---

### 2.11 Просмотр заказов (View Orders)

**Актёры:** Admin  
**Описание:** Позволяет администратору просматривать все заказы.  
**Предусловия:** Администратор вошёл в систему.  
**Основной поток:**
1. Администратор выбирает просмотр заказов;
2. Приложение отображает список всех заказов;
3. Вариант использования завершается.

---

### 2.12 Обновление статуса заказа (Update Order Status)

**Актёры:** Admin  
**Описание:** Позволяет администратору изменять статус заказов.  
**Предусловия:** Заказы существуют в системе.  
**Основной поток:**
1. Администратор выбирает заказ;
2. Администратор изменяет статус заказа;
3. Вариант использования завершается.
